布局 style / theme

本文是 loadlayout 里构造期样式（style / styleAttr / styleRes / theme）的唯一入口。颜色取色见「颜色与主题」；属性词典见「布局参考」。

相关文档 loadlayout 布局表属性参考 MaterialTextField 颜色与主题 MD3 设计

四个键分别做什么

键作用典型写法
styleAttr 构造函数的 defStyleAttr（主题里的「默认控件样式」） "?attr/materialButtonOutlinedStyle"、MDC_R.attr.textInputStyle
styleRes 构造函数的 defStyleRes（直接指定 style 资源） MDC_R.style.Widget_Material3_Button_IconButton
style 兼容旧写法：自动拆进 styleAttr 或 styleRes "?attr/…" → attr；@style/… / 数字 style id → res
theme 仅 ContextThemeWrapper，改子树主题，不是控件 Widget style 主题 overlay 名或 theme 资源 id

theme 用来包一层主题（例如 ThemeOverlay）。控件自己的外观（Outlined 按钮、轮廓输入框等）用 style / styleAttr / styleRes 写在控件表上，二者可以叠加，但含义不同。

资源怎么写

写法说明
?attr/name当前主题 attr，如 ?attr/materialButtonOutlinedStyle
?android:attr/name系统 attr
@attr/name / @style/name应用 attr / style 资源
@android:style/name系统 style
数字资源 IDMDC_R.style.xxx、MDC_R.attr.xxx、R.style.xxx（推荐，少一次字符串查找）

字符串引用会缓存；列表项很多时优先传数字 ID。解析失败时日志会提示 loadlayout: 无法解析 …。

构造顺序（实现）

LayoutViewFactory 在创建 View 时按下面顺序尝试，成功即返回。布局表没有完整 XML AttributeSet，style 主要靠构造参数，再叠加后续属性 setter。

顺序条件构造
1 有 styleRes，且类有四参构造 (Context, attrs, styleAttr, styleRes)
2 有 styleRes，无四参（如 MaterialTextField） 三参第三参 = styleRes
3 有 styleAttr 三参第三参 = styleAttr
4 回退 单参 (Context)（可先包 theme）

MaterialTextField 会根据 style / styleRes 里的 Widget 资源（如 OutlinedBox）初始化轮廓或填充外观，并带上内部输入框。布局表里写上对应的 style，就能得到完整的 Material 输入框。

推荐写法

MaterialButton：优先 styleAttr

import "com.google.android.material.button.MaterialButton"

-- 推荐：主题 attr
{ MaterialButton, text = "轮廓",
  styleAttr = "?attr/materialButtonOutlinedStyle" }

-- 兼容旧写法（同样解析为 styleAttr）
{ MaterialButton, text = "轮廓",
  style = "?attr/materialButtonOutlinedStyle" }

-- 数字 ID（少字符串查找）
{ MaterialButton, text = "轮廓",
  styleAttr = MDC_R.attr.materialButtonOutlinedStyle }

-- 纯 Widget style（IconButton 等）
{ MaterialButton, text = "",
  styleRes = MDC_R.style.Widget_Material3_Button_IconButton }

MaterialTextField：轮廓 / 填充输入框

import "vinx.material.textfield.MaterialTextField"

{
  MaterialTextField,
  layout_width = "match",
  layout_height = "wrap",
  hint = "文件名",  -- 浮动标签，写在组件上即可
  singleLine = true,
  TintColor = ColorUtil.getColorPrimary(),
  style = MDC_R.style.Widget_Material3_TextInputLayout_OutlinedBox,
  id = "file_name",
}

常用 style 名见 MaterialTextField。写上 OutlinedBox / FilledBox 等 style 后，会得到带轮廓（或填充）和内部输入区的完整控件。

hint 会显示为 Material 的浮动标签（外层 TextInputLayout）。在布局表里给组件写 hint = "…" 即可，和普通输入框一样用。

theme + styleAttr 叠加

-- theme 只改 Context；控件样式仍用 styleAttr / styleRes
{
  TextInputLayout,
  layout_width = "match",
  layout_height = "wrap",
  theme = "ThemeOverlay.Material3",  -- 主题 overlay，不是 Widget_*
  styleAttr = "?attr/textInputOutlinedStyle",
  {
    TextInputEditText,
    layout_width = "match",
    layout_height = "wrap",
  },
}

在 NeLuaJ+ 里更推荐直接用 MaterialTextField + style = MDC_R.style.Widget_…_OutlinedBox，少一层手写嵌套。

属性怎么叠在 style 上

  • 先靠 style / styleAttr / styleRes 定好控件「长什么样」，再写 text、hint、textSize、TintColor 等。
  • 布局里的颜色可以用 "?attr/color…"；代码里需要颜色 int 时用 this.themeUtil（见 颜色与主题）。
  • 想要轮廓输入框时，在表上写 Outlined 的 Widget style 最省事；描边颜色等还可以再用 TintColor 等微调。

对照与排查

你想做的可以这样写
轮廓按钮 styleAttr = "?attr/materialButtonOutlinedStyle"
轮廓输入框 + 浮动提示 MaterialTextField + style = MDC_R.style.Widget_…_OutlinedBox + hint = "…"
子树换一套主题叠加 控件上加 theme = …（ThemeOverlay），外观仍用上面的 style 字段
资源名不好记、想少查表 用 MDC_R.style.xxx / MDC_R.attr.xxx 数字 id
输入框看起来不像 Material 轮廓 补上 OutlinedBox 的 style；确认日志里没有「无法解析 style 资源」

补充

  • 旧布局里的 style = "?attr/…" 仍然有效，会按 attr 解析。
  • Material 按钮常用 styleAttr；输入框常用 style = MDC_R.style.Widget_…，两种都是推荐路径。
  • 列表里大量创建控件时，数字资源 ID 一般比字符串查找更轻。
