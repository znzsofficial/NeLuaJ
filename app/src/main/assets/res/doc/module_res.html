res 模块

res 提供类似 Android 资源系统的访问方式，支持多语言、昼夜颜色、屏幕方向尺寸，以及 drawable / font / layout / raw 等资源加载。

在 Activity 脚本中通常可直接使用全局 res；也可 require "res"（模块会把表注入环境）。

项目或 App assets:

1. res.string

多语言字符串。加载顺序：

  1. res/string/init.lua（可选，公共字符串）
  2. 设备语言：优先 语言-r地区（如 zh-rCN），再 语言（如 zh）
  3. res/string/default.lua 返回的默认语言标签（如 "en"）
  4. 最终兜底 en
-- res/string/default.lua
return "en"

-- res/string/zh.lua
hello = "你好"
app_name = "示例"

-- 使用
print(res.string.hello)
print(res.string.app_name)

2. res.language

当前实际采用的语言标签字符串，例如 "zh"、"zh-rCN"、"en"。

print(res.language)
if res.language:find("zh") then
  -- 中文环境
end

3. res.plurals

按数量选择文案。语言选择跟随 res.string。俄语（语言标签以 ru 开头）会使用 one/few/many/other 规则。

-- res/plurals/en.lua
apple_count = {
  zero = "No apples",
  one = "1 apple",
  other = "%d apples",
}

print(res.plurals.apple_count(0))
print(res.plurals.apple_count(1))
print(res.plurals.apple_count(5))

4. res.color

仅在 Activity 上下文可用。先加载 res/color/init.lua，再按是否夜间模式加载 night.lua 或 day.lua。

Material 主题色一般通过布局里的 ?attr/… 或运行时的 themeUtil 获取；res.color 更适合工程自定义色值（见 颜色与主题）。

取色 颜色与主题 
-- res/color/day.lua
accent = 0xFF1976D2
chip_bg = 0xFFE3F2FD

view.setBackgroundColor(res.color.chip_bg)

5. res.dimen

仅在 Activity 上下文可用。先 init.lua，再按方向加载 port.lua / land.lua。

-- res/dimen/port.lua
title_size = "20sp"
padding = "16dp"

print(res.dimen.title_size)

6. res.drawable

加载图标 / 图片。结果有缓存。支持两种来源（优先级如下）。

查找顺序（当前实现）

  1. Android 原生资源 R.drawable.<name>，再试 R.drawable.ic_<name>（矢量 XML，可 tint）
  2. 项目 assets res/drawable/<name>.<ext>（png / jpg / jpeg / gif / webp / svg / bmp / heif / heic / avif）
  3. 同名 Lua res/drawable/<name>.lua

因此同名时，app/src/main/res/drawable/file_code.xml 会优先于 assets 里的 file_code.png。App 内置图标建议用 vector XML；项目自定义位图仍可放 assets。

属性访问

imageView.setImageDrawable(res.drawable.my_icon)
-- 依次尝试 R.drawable.my_icon / ic_my_icon，再 assets 图片 / .lua

函数调用（着色或回调）

local t = this.themeUtil

-- 颜色整数：对 Drawable 着色（vector 用 DrawableCompat tint；位图用 SRC_ATOP 滤镜）
local icon = res.drawable("save", t.primary.main)
imageView.setImageDrawable(icon)

-- 文件列表常用：按类型 tint
local folderIcon = res.drawable("folder", t.primary.main)
local codeIcon = res.drawable("file_code", t.secondary.main)

-- 回调：加载后处理 Drawable
res.drawable("settings", function(icon)
  icon.setBounds(0, 0, 100, 100)
  someView.setBackground(icon)
end)

未找到资源时返回 nil。带着色参数时不会污染未着色缓存实例。

布局里工具栏图标推荐 src = res.drawable(name, color)。src = "@drawable/name" 的查找顺序与上相同，但不带颜色，需再写 iconTint。

7. res.bitmap

与 drawable 相同的 assets 查找路径，返回 Bitmap（Coil 解码）。不走 Android vector 回退。

local bmp = res.bitmap.photo
if bmp then
  imageView.setImageBitmap(bmp)
end

8. res.font

从 res/font/ 加载 .ttf / .otf，返回 Typeface，带缓存。

textView.setTypeface(res.font.code)

9. res.layout

加载 res/layout/<name>.lua，返回布局表，交给 loadlayout 使用。

local layout = res.layout.main
activity.setContentView(loadlayout(layout))
-- 或
activity.setContentView(res.layout.main)

布局取色用 "?attr/…"；图标 tint 用 res.drawable("save", this.themeUtil.primary.main)。

相关文档 颜色与主题 loadlayout 布局属性参考

10. res.view

加载并解析布局，直接返回 Android View，相当于 loadlayout(res.layout.xxx)。

activity.setContentView(res.view.main)

11. res.raw

访问 res/raw/ 下文件（按不含扩展名的文件名匹配）。

local f = res.raw.config          -- java.io.File
local text = res.raw("config")    -- 文本，默认 UTF-8
local gbk = res.raw("config", "GBK")

综合示例

require "import"
activity.dynamicColor()

local t = this.themeUtil
activity.setTitle(res.string.app_title or "App")
activity.setContentView(res.view.main)

titleView.setTextColor(t.primary.main)
titleView.setTypeface(res.font.code)
iconView.setImageDrawable(res.drawable("logo", t.primary.main))

local cfg = res.raw("config")
if cfg then
  print(res.raw("config"))
end

补充

  • res.color / res.dimen 会读当前 Activity 配置；在 Activity 脚本里用最省事。
  • drawable / bitmap / font / raw 有缓存；换了同名文件后，重启进程可看到最新内容。
  • 字符串与 plurals 的键名由你的资源文件定义。
