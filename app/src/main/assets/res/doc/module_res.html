res 模块

res 提供类似 Android 资源系统的访问方式，支持多语言、昼夜颜色、屏幕方向尺寸，以及 drawable / font / layout / raw 等资源加载。

在 Activity 脚本中通常可直接使用全局 res；也可 require "res"（模块会把表注入环境）。

目录结构示例

res/
  string/
    init.lua
    default.lua
    zh.lua
    zh-rCN.lua
    en.lua
  plurals/
    init.lua
    en.lua
  color/
    init.lua
    day.lua
    night.lua
  dimen/
    init.lua
    port.lua
    land.lua
  drawable/
    icon.png
    badge.lua
  font/
    code.ttf
  layout/
    main.lua
  raw/
    config.json

1. res.string

多语言字符串。加载顺序：

  1. res/string/init.lua（可选，公共字符串）
  2. 设备语言：优先 语言-r地区（如 zh-rCN），再 语言（如 zh）
  3. res/string/default.lua 返回的默认语言标签（如 "en"）
  4. 最终兜底 en
-- res/string/default.lua
return "en"

-- res/string/zh.lua
hello = "你好"
app_name = "示例"

-- 使用
print(res.string.hello)
print(res.string.app_name)

2. res.language

当前实际采用的语言标签字符串，例如 "zh"、"zh-rCN"、"en"。

print(res.language)
if res.language:find("zh") then
  -- 中文环境
end

3. res.plurals

按数量选择文案。语言选择跟随 res.string。俄语（语言标签以 ru 开头）会使用 one/few/many/other 规则。

-- res/plurals/en.lua
apple_count = {
  zero = "No apples",
  one = "1 apple",
  other = "%d apples",
}

-- 使用：返回对应条目字符串（是否 sprintf 取决于你的资源写法）
print(res.plurals.apple_count(0))
print(res.plurals.apple_count(1))
print(res.plurals.apple_count(5))

4. res.color

仅在 Activity 上下文可用。先加载 res/color/init.lua，再按是否夜间模式加载 night.lua 或 day.lua。

-- res/color/day.lua
primary = 0xFF1976D2
bg = 0xFFFFFFFF

-- res/color/night.lua
primary = 0xFF90CAF9
bg = 0xFF121212

-- 使用
view.setBackgroundColor(res.color.bg)

5. res.dimen

仅在 Activity 上下文可用。先 init.lua，再按方向加载 port.lua / land.lua（或 undefined）。

-- res/dimen/port.lua
title_size = "20sp"
padding = "16dp"

-- 使用（具体值类型取决于你在文件中写的内容）
print(res.dimen.title_size)

6. res.drawable

从 res/drawable/ 加载图片或同名 .lua 脚本。支持扩展名：png / jpg / jpeg / gif / webp / svg / bmp / heif / heic / avif。结果有缓存。

属性访问

imageView.setImageDrawable(res.drawable.my_icon)
-- 查找 res/drawable/my_icon.png 等，或 res/drawable/my_icon.lua

函数调用（可着色或回调）

-- 颜色整数：对 Drawable 应用 SRC_ATOP 颜色滤镜
local redIcon = res.drawable("my_icon", 0xFFFF0000)
imageView.setImageDrawable(redIcon)

-- 回调：加载后处理 Drawable
res.drawable("settings_icon", function(icon)
  icon.setBounds(0, 0, 100, 100)
  someView.setBackground(icon)
end)

未找到资源时返回 nil。

7. res.bitmap

与 drawable 相同查找路径，但返回 Bitmap（经 Coil 解码）。

local bmp = res.bitmap.photo
if bmp then
  imageView.setImageBitmap(bmp)
end

8. res.font

从 res/font/ 加载 .ttf / .otf，返回 Typeface，带缓存。

textView.setTypeface(res.font.code)
-- 对应 res/font/code.ttf 或 code.otf

9. res.layout

加载 res/layout/<name>.lua，返回布局表（table），交给 loadlayout 使用。

local layout = res.layout.main
activity.setContentView(loadlayout(layout))

10. res.view

加载并解析布局，直接返回 Android View，相当于 loadlayout(res.layout.xxx) 的快捷方式。

local root = res.view.main
activity.setContentView(root)

11. res.raw

访问 res/raw/ 下任意文件（按文件名不含扩展名匹配）。

-- 属性访问：返回 java.io.File
local f = res.raw.config
print(f.getAbsolutePath())

-- 函数调用：读取文本内容，默认 UTF-8
local text = res.raw("config")
local gbkText = res.raw("config", "GBK")
print(text)

综合示例

activity.setTitle(res.string.app_name)
activity.setContentView(res.view.main)

titleView.setTextColor(res.color.primary)
titleView.setTypeface(res.font.code)
iconView.setImageDrawable(res.drawable("logo", res.color.primary))

local cfg = res.raw("config")
if cfg then
  print(res.raw("config"))
end

注意

  • res.color / res.dimen 依赖 Activity 配置；非 Activity 上下文可能不可用。
  • drawable / bitmap / font / raw 都有缓存，同名资源热替换后可能需重启进程。
  • 字符串与 plurals 的具体键名完全由你的资源文件定义。
