1. dynamicColor

会优先使用设置页保存的全局种子 theme_seed_color；若未设置，则在支持的系统上跟随壁纸取色（Android 12+）。

-- 通常在 main.lua 里、setContentView 之前
activity.dynamicColor()
-- 也可以传入临时种子色（会覆盖用户在设置里选的全局种子）
-- activity.dynamicColor(0xFF6750A4)

activity.setContentView(res.layout.main)
  • 在 setContentView 之前调用时，后续布局才能用到新的动态色板。
  • 设置里已经保存了 theme_seed_color 时，页面里调用 dynamicColor() 即可，一般不必再自己读 SharedData 拼种子。

2. 布局：?attr/…

loadlayout 创建 View 时会解析主题属性，写法接近 Android XML。工程模板里的布局也多采用这种方式。

return {
  LinearLayout,
  BackgroundColor = "?attr/colorBackground",
  {
    MaterialTextView,
    textColor = "?attr/colorPrimary",
  },
  {
    MaterialCardView,
    CardBackgroundColor = "?attr/colorSurfaceContainer",
    strokeColor = "?attr/colorOutlineVariant",
    strokeWidth = "1dp",
    radius = "16dp",
  },
}

常用 attr：

attr用途
colorPrimary / colorOnPrimary主色 / 其上内容
colorPrimaryContainer / colorOnPrimaryContainer主色容器
colorSecondary / colorSecondaryContainer次色
colorSurface / colorOnSurface表面 / 正文
colorOnSurfaceVariant次要文字
colorSurfaceContainer（及 Low / High / Highest）卡片、列表
colorBackground / colorOnBackground页面背景
colorOutline / colorOutlineVariant描边
colorError错误态
actionBarSize工具栏高度（非颜色）

也支持 ?android:attr/…。

3. 运行时：themeUtil

需要 int 时用（图标 tint、动态 setBackgroundColor、条件着色等）。

local t = this.themeUtil
res.drawable("save", t.primary.main)
view.setBackgroundColor(t.surface.container)

-- 分组速览（完整表 → LuaThemeUtil.html）
-- t.primary / secondary / tertiary / error
-- t.surface / background / outline / text

旧写法 t.getColorPrimary() 仍可用，新代码请用 t.primary.main。

-- 任意 attr
local c = t.getAnyColor(MDC_R.attr.colorTertiaryContainer)

4. 其它工具

-- 与主题 primary 协调
local themed = this.harmonizeColor(0xFFFF0000)

local mixed = this.harmonizeColor(c1, c2)

-- 明暗（决定文字黑/白）
if this.isColorLight(bg) then
  text.setTextColor(0xFF000000)
else
  text.setTextColor(0xFFFFFFFF)
end
