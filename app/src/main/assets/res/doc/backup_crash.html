异常捕获与代码备份

编辑器在保存代码时会留下快照；未处理的崩溃会写入日志文件。这些内容都在应用的媒体目录（以及可选的工程 ZIP 备份目录）中，方便事后查看和恢复。

崩溃日志

未捕获异常默认写入：

/sdcard/Android/media/github.znzsofficial.neluaj/crash/
  • 文件名里通常带有时间信息，便于按时间排序。
  • 内容一般包括异常类型、消息和堆栈，有助于对应到脚本行号。
  • 在系统文件管理器中打开上述路径即可浏览（设备与权限情况因机而异）。

脚本里可以定义 onError，自定义如何处理错误；返回值还会影响默认日志展示方式。说明见 LuaActivity 中的错误处理部分。

保存时的代码备份

编辑器每次成功保存后，会在下面目录再写一份副本（按日期分子目录）：

/sdcard/Android/media/github.znzsofficial.neluaj/backups/
  • 适合误覆盖、误删后对照恢复。
  • 与工程目录相互独立：删除工程文件夹不会自动清空这里的备份。
  • 占用空间会随时间增长，需要时可以自行清理较旧的文件。

在 IDE 中浏览

主界面菜单 工具 → Media 浏览器 可直接打开相关目录，例如：

  • crash/ — 崩溃日志，以及 vConsole 导出的文本
  • backups/ — 保存时的代码快照
  • /sdcard/LuaJ/Backup/ — 菜单「备份」生成的工程 ZIP

浏览器支持查看路径、预览文本、复制路径、分享和删除等操作。

使用上的提示

  • 发布或打包前，可以打开 crash/ 看是否还有近期未处理的崩溃记录。
  • 大改代码前，可以用菜单「备份」再存一份完整工程 ZIP。
  • 调试时，在 onError 里补充当前文件路径、关键变量等上下文，日志会更好读。
