init.lua configuration

init.lua lives in the project root and describes app name, package, version, theme, permissions, and related environment fields. The host Activity reads these values at startup. You can also generate the file with the official builder and adjust it as needed.

Each field below includes its type, meaning, and a short example.

1. app_name

  • Type: string
  • Meaning: Display name of the app; also used as a default window title source.
app_name = "Sample Application"

2. package_name

  • Type: string
  • Meaning: Application id / package name used to identify the app. Common form: com.example.myapp (lowercase, dot-separated).
package_name = "com.example.myapp"

3. ver_name

  • Type: string
  • Meaning: User-visible version label, e.g. 1.0.0.
ver_name = "1.0.0"

4. ver_code

  • Type: integer
  • Meaning: Internal version code, usually increasing with each release so builds can be compared.
ver_code = 101

5. debug_mode

  • Type: boolean
  • Meaning: When true, the runtime may surface more debug information (for example via Toast).
  • vConsole: The floating debugger is a separate module. Load it with require "vConsole" when you need it. If you enable the vConsole module while creating a project, the file is copied to the project root and main.lua gets the matching require.
debug_mode = true

6. target_sdk

  • Type: integer
  • Meaning: Target Android SDK level the app is designed for. This is usually aligned with the target used when packaging, e.g. 33.
target_sdk = 33

7. min_sdk

  • Type: integer
  • Meaning: Lowest Android SDK level the app declares support for. Devices below this level may not install or run the app.
min_sdk = 24

8. NeLuaJ_Theme

  • Type: string
  • Meaning: Activity theme name matching a field on the app’s R.style. Applied when the Activity starts, so activity.setTheme() in main.lua is usually unnecessary.

Dots in style names become underscores in the string, e.g. Theme.NeLuaJ.Material3 → Theme_NeLuaJ_Material3.

ValueDescription
Material 3
Theme_NeLuaJ_Material3MD3 with ActionBar; light/dark follows the system
Theme_NeLuaJ_Material3_ActionOverlayMD3 + ActionBar + ActionMode overlay
Theme_NeLuaJ_Material3_NoActionBarMD3 without ActionBar (custom Toolbar)
Theme_NeLuaJ_Material3_NoActionBar_ActionOverlayMD3 without ActionBar + ActionMode overlay
Material 3 Dynamic Colors
Theme_NeLuaJ_Material3_DynamicColors (and variants)MD3 with wallpaper/user seed colors
AppCompat / Material Components (MD2)
Theme_NeLuaJ_Compat… / Theme_NeLuaJ_MaterialComponent…Legacy DayNight or fixed light/dark themes

Project look-and-feel is set with NeLuaJ_Theme. New-project main.lua does not call activity.dynamicColor() by default; add it at the entry point when you want Material You / a seed palette. How that interacts with the Settings accent color is described in Colors and theme.

NeLuaJ_Theme = "Theme_NeLuaJ_Material3_NoActionBar_ActionOverlay"

9. theme (legacy)

  • Type: integer or string
  • Meaning: Older theme field. Prefer NeLuaJ_Theme for app R.style themes (including Material 3). If you still use theme: a number is a style resource id; a string is looked up on android.R.style only—platform themes such as Theme_DeviceDefault_DayNight or Theme_Material_Light, not Material 3 names.
theme = "Theme_DeviceDefault_DayNight"
-- theme = "Theme_Material_Light"

10. user_permission

  • Type: table (array of strings)
  • Meaning: Android permission names the app may need. Packaging and runtime permission handling use this list. Names match Android’s permission constants, e.g. android.permission.INTERNET.
user_permission = {
  "android.permission.INTERNET",
  "android.permission.CAMERA",
}

Full example

app_name = "Sample Application"
package_name = "com.example.myapp"
ver_name = "1.0.0"
ver_code = 101
debug_mode = true
target_sdk = 33
min_sdk = 24
NeLuaJ_Theme = "Theme_NeLuaJ_Material3"
user_permission = {
  "android.permission.INTERNET",
  "android.permission.CAMERA",
}

Notes

  • init.lua is normally next to main.lua at the project root; the runtime looks for configuration there.
  • The file is a simple Lua assignment script; syntax errors prevent loading and usually appear in the log.
  • Permission strings that do not match Android’s names may not be requested or packaged as expected.
