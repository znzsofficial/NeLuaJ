LuaActivity 完整文档

LuaActivity 是 NeLuaJ+ 的核心宿主 Activity，继承自 AppCompatActivity。所有 Lua 脚本都运行在它的上下文中。

全局变量 this 和 activity 都指向当前 LuaActivity 实例。

1. 生命周期回调

在 Lua 脚本中定义以下函数，Activity 会在对应时机自动调用：

function onCreate()     end  -- Activity 创建完成，布局已加载
function onStart()      end  -- Activity 可见
function onResume()     end  -- Activity 获得焦点
function onPause()      end  -- Activity 失去焦点
function onStop()       end  -- Activity 不可见
function onDestroy()    end  -- Activity 销毁

Activity 结果

Lua 页面之间返回数据推荐使用 newActivity + onResult。系统页面、文件选择、设置页等 Android Intent 推荐使用 resultLauncher。完整示例见“Activity 结果使用”。

-- 方式一：Lua 页面之间返回结果（推荐）
this.newActivity(1001, "other.lua", { "参数" })

function onResult(name, ...)
    print("返回自:", name)
    print("返回值:", ...)
    -- return true 可拦截后续 onActivityResult
end

-- 兼容旧脚本和需要原始 Android 参数的场景
function onActivityResult(requestCode, resultCode, data)
    print(requestCode, resultCode, data)
end

-- 共享元素返回 / Activity reenter（API 21+）
-- 从带共享元素动画的二级页返回时调用，可在此读取 data 并重新映射过渡
function onActivityReenter(resultCode, data)
    print(resultCode, data)
end

错误处理

--[[
  报错时调用，返回值控制输出方式：
  "trace"   — 完整堆栈
  "log"     — 仅日志
  "message" — 仅消息
  "title"   — 仅标题
  返回其他非空值可拦截默认报错
]]
function onError(error, message)
    print(error, message)
    return true
end

7. 权限管理

权限相关 API 分为三类：单个权限状态检查、批量检查并请求 Manifest 中声明的危险权限、存储权限专用流程。普通危险权限推荐用 permissionLauncher 或 checkAllPermissions；存储权限推荐用 requestStoragePermission。

单个权限检查

-- 检查任意权限是否已授权
local granted = this.checkPermission("android.permission.CAMERA")
if granted then
    print("相机权限已授权")
else
    print("相机权限未授权")
end

请求单个权限

-- 创建权限请求 Launcher
local cameraPermission = this.permissionLauncher(function(granted)
    print("相机权限结果:", granted)
end)

cameraPermission.launch("android.permission.CAMERA")

检查并请求 Manifest 危险权限

checkAllPermissions 会扫描应用 Manifest 中声明的危险权限，自动请求尚未授权的部分。返回 true 表示当前已经全部授权或没有需要请求的危险权限；返回 false 表示已经发起系统权限请求，结果会进入 onRequestPermissionsResult。

if this.checkAllPermissions() then
    print("所有危险权限都已满足")
else
    print("已发起权限请求，等待用户选择")
end

function onRequestPermissionsResult(requestCode, permissions, grantResults)
    print("权限请求完成:", requestCode)
end

存储权限

requestStoragePermission 会根据 Android 版本选择正确流程：Android 11 及以上跳转所有文件访问权限设置页，Android 10 及以下请求传统外部存储读取权限。结果通过 onStorageRequestResult(isGranted) 返回。

if this.checkStoragePermission() then
    print("已有存储权限")
else
    this.requestStoragePermission()
end

function onStorageRequestResult(isGranted)
    print("存储权限结果:", isGranted)
end

8. 主题与颜色

主题设置

Activity 启动时会自动读取 init.lua 中的主题配置：

-- init.lua（启动时读取并应用）
NeLuaJ_Theme = "Theme_NeLuaJ_Material3"                                    -- MD3 + ActionBar
NeLuaJ_Theme = "Theme_NeLuaJ_Material3_NoActionBar_ActionOverlay"         -- 无 ActionBar
NeLuaJ_Theme = "Theme_NeLuaJ_Material3_DynamicColors_NoActionBar_ActionOverlay" -- 动态取色

-- 旧字段 theme：仅 android.R.style 平台主题名，或样式资源 ID
-- （Material 3 主题名不在 android.R.style 中，请用上面的 NeLuaJ_Theme）
theme = "Theme_DeviceDefault_DayNight"
theme = "Theme_Material_Light"
-- theme = 0x01030237  -- 整数资源 ID

完整主题名列表见 init.lua 配置说明。

若要在脚本里改主题，可在创建内容视图之前调用：

activity.setTheme(R.style.Theme_NeLuaJ_Material3_NoActionBar_ActionOverlay)

动态颜色与深色模式

如果应用已经保存了全局主题色种子 theme_seed_color，启动时会优先用该值生成整套动态色板。也就是说，绝大多数页面只需要继续调用 this.dynamicColor()，无需每个页面重复处理种子色逻辑。

设置页提供了预设色板和自定义颜色入口，最终会把结果保存到 theme_seed_color。后续进入应用的页面会沿用这个值，从而保持整个软件的配色一致。

-- 启用 Material You 动态颜色
this.dynamicColor()

-- 判断深色模式
if this.isNightMode() then
    print("当前为深色模式")
end

主题颜色工具

-- 取色：布局用 ?attr/，运行时用 themeUtil（见 color_api.html）
local t = this.themeUtil
local filter = this.getFilter(t.primary.main)
drawable.setColorFilter(filter)

13. Activity 结果使用

结果回调分三类：Lua 页面之间返回值用 newActivity + this.result + onResult；启动系统页面或第三方 Activity 用 resultLauncher；单个权限请求用 permissionLauncher。旧式 onActivityResult 仍保留为兼容入口，但新代码优先使用前两种方式。

13.1 Lua 页面返回结果

调用方使用带请求码的 newActivity 打开另一个 Lua 文件，被打开页面调用 this.result 返回数组。返回后会先触发 onResult，如果 onResult 返回 true，则不会继续触发 onActivityResult。

-- main.lua：打开 other.lua，并等待返回值
this.newActivity(1001, "other.lua", { "传给 other 的参数" })

function onResult(name, ...)
    print("返回页面:", name)       -- 通常是 "other.lua"
    print("返回值:", ...)         -- this.result 传回的数组会展开
    return true                  -- 已处理，拦截 onActivityResult
end

function onActivityResult(requestCode, resultCode, data)
    print("原始结果:", requestCode, resultCode, data)
end

-- other.lua：返回结果并关闭当前页面
function onCreate()
    this.result({ "保存成功", 123, true })
end

this.result 会使用 Activity.RESULT_OK 作为返回码。只需要业务返回值时，直接处理 onResult 即可。

13.2 启动系统页面或第三方 Activity

resultLauncher 是 Android Activity Result API 的封装，用于替代直接调用 startActivityForResult。推荐在 onCreate 或脚本初始化阶段创建 launcher，然后在按钮点击、菜单点击等交互中调用 launch。

-- 选择一张图片
import "android.app.Activity"
import "android.content.Intent"

local pickImageLauncher = this.resultLauncher(function(result)
    if result.getResultCode() == Activity.RESULT_OK then
        local data = result.getData()
        local uri = data and data.getData()
        print("选择的图片:", uri)
    else
        print("用户取消选择")
    end
end)

function pickImage()
    local intent = Intent(Intent.ACTION_GET_CONTENT)
    intent.setType("image/*")
    pickImageLauncher.launch(intent)
end

启动 Android 设置页、文档选择器、联系人选择器等都可以使用同一个模式：先创建 Intent，再调用 launcher.launch(intent)。

-- 打开应用详情设置页，返回后重新检查状态
import "android.app.Activity"
import "android.content.Intent"
import "android.net.Uri"
import "android.provider.Settings"

local settingsLauncher = this.resultLauncher(function(result)
    print("设置页已返回:", result.getResultCode())
end)

local intent = Intent(Settings.ACTION_APPLICATION_DETAILS_SETTINGS)
intent.setData(Uri.parse("package:" .. this.getPackageName()))
settingsLauncher.launch(intent)

13.3 权限请求

请求单个权限使用 permissionLauncher。如果需要存储权限，优先使用 requestStoragePermission，它会按 Android 版本自动选择正确流程。

-- 请求相机权限
local cameraPermission = this.permissionLauncher(function(granted)
    if granted then
        print("相机权限已授权")
    else
        print("相机权限被拒绝")
    end
end)

cameraPermission.launch("android.permission.CAMERA")

-- 请求存储权限（推荐使用封装方法）
this.requestStoragePermission()

function onStorageRequestResult(granted)
    print("存储权限:", granted)
end

13.4 旧式 onActivityResult 兼容

旧脚本如果直接调用 startActivityForResult，仍可通过 onActivityResult 接收结果。新代码不建议继续依赖这种方式，因为它需要手动维护 requestCode，也不如 launcher 方式清晰。

-- 兼容旧写法：能用，但新代码更推荐 resultLauncher
import "android.content.Intent"

local intent = Intent(Intent.ACTION_GET_CONTENT)
intent.setType("image/*")
this.startActivityForResult(intent, 2001)

function onActivityResult(requestCode, resultCode, data)
    if requestCode == 2001 then
        print("旧式回调:", resultCode, data and data.getData())
    end
end

13.5 常见注意点

  • onResult(name, ...) 适合 Lua 页面之间传业务数据，参数来自被打开页面的 this.result({ ... })。
  • onActivityResult(requestCode, resultCode, data) 适合兼容旧脚本，或者确实需要原始 Android Intent 数据。
  • onActivityReenter(resultCode, data) 在共享元素过渡返回本页时调用（API 21+），可在此读取 data 并配合 ActivityOptions 做 reenter 映射；与 onActivityResult 独立，不是替代关系。
  • onResult 返回 true 表示结果已处理，会阻止继续调用 onActivityResult。
  • this.newActivity("other.lua", true) 是多文档模式，会创建独立任务栈，不会返回 Activity 结果。
  • resultLauncher 和 permissionLauncher 可以先保存为局部变量或全局变量，再在点击事件中调用 launch。
  • 启动系统页面时要根据 result.getResultCode() 判断用户是否完成操作；文件选择类结果通常从 result.getData().getData() 取 Uri。
