init.lua 配置说明

init.lua 放在工程根目录，用来描述应用名称、包名、版本、主题、权限等环境信息。Activity 启动时会读取这些字段。也可以用官方打包器生成，再按需微调。

下面按字段说明类型、含义与示例。

1. app_name

  • 类型：字符串
  • 含义：应用显示名称，也会用作窗口标题的默认来源之一。
app_name = "示例应用"

2. package_name

  • 类型：字符串
  • 含义：应用包名，用于在系统中区分应用。常见写法类似 com.example.myapp（小写、点分段）。
package_name = "com.example.myapp"

3. ver_name

  • 类型：字符串
  • 含义：面向用户的版本名，例如 1.0.0。
ver_name = "1.0.0"

4. ver_code

  • 类型：整数
  • 含义：内部版本号，一般随发布递增，便于比较构建新旧。
ver_code = 101

5. debug_mode

  • 类型：布尔值
  • 含义：为 true 时，运行环境会更积极地通过 Toast 等方式展示调试信息。
  • 与 vConsole：悬浮调试窗是独立模块。需要时在脚本中 require "vConsole"；创建项目时若勾选了 vConsole 模块，会把文件放进工程根目录，并在 main.lua 中写上对应 require。
debug_mode = true

6. target_sdk

  • 类型：整数
  • 含义：目标 Android SDK 级别，影响系统按哪个 API 行为运行应用。通常与打包/构建时的目标版本一致，例如 33。
target_sdk = 33

7. min_sdk

  • 类型：整数
  • 含义：应用声明支持的最低 Android SDK 级别。低于该级别的设备可能无法安装或运行。
min_sdk = 24

8. NeLuaJ_Theme

  • 类型：字符串
  • 含义：Activity 主题名，对应应用内 R.style 中的字段。启动时会按该名称应用主题，一般不必再在 main.lua 里调用 activity.setTheme()。

字符串里的点号写成下划线，例如样式 Theme.NeLuaJ.Material3 对应 Theme_NeLuaJ_Material3。

值说明
Material 3
Theme_NeLuaJ_Material3MD3，带 ActionBar；随系统日/夜切换色板
Theme_NeLuaJ_Material3_ActionOverlayMD3 + ActionBar + ActionMode 覆盖
Theme_NeLuaJ_Material3_NoActionBarMD3，无 ActionBar（适合自定义 Toolbar）
Theme_NeLuaJ_Material3_NoActionBar_ActionOverlayMD3，无 ActionBar + ActionMode 覆盖
Material 3 Dynamic Colors
Theme_NeLuaJ_Material3_DynamicColorsMD3 动态取色，带 ActionBar
Theme_NeLuaJ_Material3_DynamicColors_ActionOverlay动态取色 + ActionBar + ActionMode 覆盖
Theme_NeLuaJ_Material3_DynamicColors_NoActionBar动态取色，无 ActionBar
Theme_NeLuaJ_Material3_DynamicColors_NoActionBar_ActionOverlay动态取色，无 ActionBar + ActionMode 覆盖
AppCompat
Theme_NeLuaJ_CompatAppCompat DayNight，带 ActionBar
Theme_NeLuaJ_Compat_NoActionBarAppCompat DayNight，无 ActionBar
Material Components（MD2）
Theme_NeLuaJ_MaterialComponentMD2 DayNight，带 ActionBar
Theme_NeLuaJ_MaterialComponent_NoActionBarMD2 DayNight，无 ActionBar
Theme_NeLuaJ_MaterialComponent_Light / _NoActionBarMD2 固定浅色
Theme_NeLuaJ_MaterialComponent_Dark / _NoActionBarMD2 固定深色

动态取色主题常与 activity.dynamicColor() 或设置页中的主题种子色一起使用，界面会更贴近壁纸/用户所选色板。详见 颜色与主题。

NeLuaJ_Theme = "Theme_NeLuaJ_Material3_NoActionBar_ActionOverlay"

9. theme（旧字段）

  • 类型：整数或字符串
  • 含义：较早版本用来指定主题的字段。现在更常见的是写 NeLuaJ_Theme（应用 R.style 中的 NeLuaJ / Material 主题）。若仍使用本字段：数字表示主题资源 ID；字符串会从 android.R.style 按字段名查找——只有平台自带主题（如 Theme_DeviceDefault_DayNight、Theme_Material_Light），没有 Material 3 名称。
theme = "Theme_DeviceDefault_DayNight"
-- theme = "Theme_Material_Light"

10. user_permission

  • 类型：表（字符串数组）
  • 含义：声明应用可能用到的 Android 权限名列表。打包与运行时会按此列表处理权限相关逻辑。名称与 Android 官方权限字符串一致，例如 android.permission.INTERNET。
user_permission = {
  "android.permission.INTERNET",
  "android.permission.CAMERA",
}

完整示例

app_name = "示例应用"
package_name = "com.example.myapp"
ver_name = "1.0.0"
ver_code = 101
debug_mode = true
target_sdk = 33
min_sdk = 24
NeLuaJ_Theme = "Theme_NeLuaJ_Material3"
user_permission = {
  "android.permission.INTERNET",
  "android.permission.CAMERA",
}

补充说明

  • init.lua 与 main.lua 一般同级，都在工程根目录；运行时按工程根查找配置。
  • 文件是普通 Lua 赋值脚本，语法错误会导致配置加载失败，可从日志中查看提示。
  • 权限名拼写与系统不一致时，对应权限可能无法按预期申请或打包。
