lazy 懒加载

lazy 用于延迟计算：包装一个函数或带参数的调用，只有在读取 .value 时才真正执行，之后会缓存结果。

基本用法

-- 无参方法
local media = lazy(this.getMediaDir)

-- 带参数方法：第二个参数起作为调用参数
local sub = lazy(this.getLuaDir, "subdir")

-- 匿名函数
local v = lazy(function()
  return heavyWork()
end)

-- LuaJ++ lambda
local w = lazy(lambda(): "value")

print(media.value) -- 首次访问时计算
print(media.value) -- 直接使用缓存

行为说明

  • 第一次访问 .value 时执行包装的逻辑。
  • 结果会被缓存；再次访问不会重复执行（除非实现另行重置）。
  • 适合初始化成本高、但未必每次都用到的路径、配置或对象。

适用场景

  • 媒体目录、缓存目录等路径解析。
  • 可选模块的延迟加载。
  • 只在用户进入某页面时才需要的数据准备。

注意

  • 包装的函数若依赖 UI 状态，应确保首次 .value 访问时上下文仍然有效。
  • 若函数会抛错，错误会在首次访问时暴露，而不是创建 lazy 时。
