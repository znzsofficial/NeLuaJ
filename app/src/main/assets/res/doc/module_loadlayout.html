介绍

使用 LuaLayout 加载布局表的函数。

支持更多属性，例如 textStyle、layout_anchor、pages（for ViewPager）等，并支持 Material 组件的主题/样式增强：theme、style、styleAttr、styleRes。

参数类型分别为 LuaTable、LuaTable、ViewGroup.LayoutParams。

loadlayout(layout)
loadlayout(layout, _G)
loadlayout(layout, _G, FrameLayout.LayoutParams)

基本结构

布局表的第一个元素是 View 类（例如 LinearLayout、MaterialTextView）。后面的字符串键表示属性，数字键表示子视图。

return {
	LinearLayout,
	orientation = "vertical",
	layout_width = "match",
	layout_height = "match",
	{
		TextView,
		id = "title",
		text = "标题",
		textSize = "20sp",
	},
}

padding / margin 与 Start·End

LuaLayout 在属性循环结束后统一解析间距（不会当作普通 setter 漏设）：

写法实现说明
padding 与 paddingLeft/Top/Right/BottomView.setPadding物理方向；单边可覆盖 padding
含 paddingStart 或 paddingEndView.setPaddingRelativeRTL 感知；缺省边回退 Left/Right/padding
layout_margin 与 Left/Top/Right/BottomLayoutParams.setMargins四边外边距
layout_marginStart / layout_marginEndsetMarginStart / setMarginEnd在 setMargins 之后写入，避免被覆盖
return {
  LinearLayout,
  orientation = "vertical",
  paddingStart = "16dp",
  paddingEnd = "16dp",
  {
    MaterialButton,
    text = "确定",
    layout_marginStart = "8dp",
    layout_marginEnd = "8dp",
  },
}

事件监听

所有以 on 开头的属性会按监听器处理。属性值可以是函数，也可以是绑定表中的函数名字符串。

local binding = {
	onButtonClick = function(view)
		print("clicked")
	end,
}

local view = loadlayout({
	Button,
	text = "点击",
	onClick = "onButtonClick",
}, binding)

{
	Button,
	text = "直接函数",
	onClick = function(view)
		print(view)
	end,
}

style / theme 用法

loadlayout 会在创建 View 时优先处理构造期样式字段，适合 MaterialButton、MaterialTextView、TextInputLayout、MaterialCardView 等组件。

兼容字段

  • style：旧写法，仍可继续使用；传 ?attr/... 时会按 defStyleAttr 处理，传 @style/... 或样式资源 ID 时按旧 style 兼容。
  • theme：视图创建时使用的主题资源，会包装当前 Context。
  • styleAttr：构造函数的 defStyleAttr，推荐用于 Material 组件的变体样式。
  • styleRes：构造函数的 defStyleRes，可与 styleAttr 组合使用。

支持的资源写法：

写法说明
?attr/name当前主题中的属性，例如 ?attr/materialButtonOutlinedStyle、?attr/colorPrimary
?android:attr/nameAndroid 系统 attr
@attr/name应用 attr 资源
@style/name应用 style 资源
@android:style/nameAndroid 系统 style
数字资源 ID直接使用 R.attr.xxx / R.style.xxx / MDC_R.attr.xxx

颜色属性

颜色字段推荐 "?attr/colorPrimary" 等（默认模板）。

{ MaterialTextView, textColor = "?attr/colorPrimary" }

示例

旧写法仍然可用：

import "com.google.android.material.button.MaterialButton"

return {
	MaterialButton,
	text = "轮廓按钮",
	style = "?attr/materialButtonOutlinedStyle",
}

推荐写法：显式区分 styleAttr 与 styleRes。

import "com.google.android.material.button.MaterialButton"

return {
	MaterialButton,
	text = "轮廓按钮",
	styleAttr = "?attr/materialButtonOutlinedStyle",
}

同时指定 theme overlay 与样式：

import "com.google.android.material.textfield.TextInputLayout"
import "com.google.android.material.textfield.TextInputEditText"

return {
	TextInputLayout,
	layout_width = "match",
	layout_height = "wrap",
	theme = "ThemeOverlay.Material3",
	styleAttr = "?attr/textInputOutlinedStyle",
	{
		TextInputEditText,
		layout_width = "match",
		layout_height = "wrap",
	}
}

如果已经拿到资源 ID，优先传数字 ID，可避免运行时字符串查找：

return {
	MaterialButton,
	text = "高性能写法",
	styleAttr = MDC_R.attr.materialButtonOutlinedStyle,
}

说明

  • style 仍可用于既有布局。
  • Material 组件上 styleAttr / styleRes 较常见；也可再叠加 theme。
  • 字符串形式的资源引用会做缓存；列表项很多时，直接传数字资源 ID 通常更省开销。
  • 资源名无效时，运行日志里会给出较明确的提示，便于对照布局表排查。

CoordinatorLayout 特殊属性

loadlayout 对部分 Material/CoordinatorLayout 布局参数做了特殊处理：

属性说明示例
layout_behavior设置 CoordinatorLayout Behavior"@string/appbar_scrolling_view_behavior"
layout_anchor锚定到已定义 id 的 View"fab"
layout_scrollFlagsAppBar 滚动标志`"scroll
layout_collapseModeCollapsingToolbar 折叠模式"pin", "parallax"
layout_collapseParallaxMultiplier视差系数0.5

相对布局规则也支持字符串 id 引用，例如 layout_below = "title"。被引用的 id 需要在当前 View 之前定义。

补充说明

  • 第一个元素若是 nil，常见原因是尚未 import 对应类，或类名写错。
  • id = "title" 会给 View 分配 Android ID，并把该 View 放进绑定表（如 binding.title）。
  • style / styleAttr / styleRes / theme 只在构造 View 时生效，不会再当普通属性写一遍。
  • padding* 与 layout_margin*（含 Start/End）在整表属性扫完后统一应用，详见上文「padding / margin 与 Start·End」。
  • 多数 Java setXxx 可以用同名属性写，例如 alpha = 0.5、enabled = false。MaterialButton 等若默认 style 盖住了 tint，可在 loadlayout 返回后再调用一次 setBackgroundTintList。
  • 属性名对不上任何 setter / public field 时，会通过错误日志提示（例如拼写错误）。专用键如 id、padding*、pages、style* 按各自逻辑处理；onClick 这类监听器按约定绑定。
  • 字符串枚举按属性分别解析：gravity、inputType、imeOptions、visibility 等各有一套取值。多个 flag 可用 | 连接，例如 gravity = "center|end"。尺寸关键字 match / wrap / fill 在宽高里通用。
  • textSize：带 sp/dp 等单位时按单位换算；纯数字按 sp 理解，再以像素写入。
  • minWidth / minHeight 支持带 dp 的字符串。
  • layout_width / layout_height 支持 match/wrap 与尺寸字符串；外边距写入 LayoutParams 的 margin 字段；单边 margin 只改写到的边。
  • 子节点可以是嵌套表，也可以是字符串路径（会先 require 再加载）。
