loadlayout

把 Lua 布局表转成 Android View。本文讲用法与机制（参数、图片 src、样式、事件等）。完整属性词典请打开布局参考。

loadlayout(layout)
loadlayout(layout, env)                       -- env：id 绑定表，常用 _G 或 binding
loadlayout(layout, env, FrameLayout.LayoutParams)  -- 可选：根 LayoutParams 类型

基本结构

布局表的第一个元素是 View 类（例如 LinearLayout、MaterialTextView）。后面的字符串键表示属性，数字键表示子视图。

return {
	LinearLayout,
	orientation = "vertical",
	layout_width = "match",
	layout_height = "match",
	{
		TextView,
		id = "title",
		text = "标题",
		textSize = "20sp",
	},
}

参数说明

参数类型说明
layoutLuaTable布局表，或可被 require 得到的布局表
envLuaTable可选。绑定 id 对应 View 的表，常用 _G 或局部 binding
paramsViewGroup.LayoutParams 类可选。指定根视图 LayoutParams 类型
local binding = {}
local view = loadlayout(layout, binding)
print(binding.title) -- id = "title" 的 View

src 图片

给 ImageView / AppCompatImageView 设图。工具栏、列表图标推荐用 res.drawable 一次完成「找图 + 上色」。

场景推荐写法
图标 + 主题色 src = res.drawable("save", ColorUtil.getColorPrimary())
只要应用内矢量，颜色另设 src = "@drawable/ic_copy"，再写 iconTint = …
本地文件 / 网络图 src = "/sdcard/a.png" 或 URL（异步加载） 
-- 推荐：加载并着色（name 可写短名 save，会再试 ic_save）
{
  AppCompatImageView,
  layout_width = "40dp",
  layout_height = "40dp",
  padding = "8dp",
  src = res.drawable("save", ColorUtil.getColorPrimary()),
}

-- 资源引用本身不着色，需另写 iconTint
{
  AppCompatImageView,
  src = "@drawable/ic_copy",
  iconTint = ColorUtil.getColorPrimary(),
}

-- 路径 / URL（异步）
{
  ImageView,
  src = "https://example.com/a.png",
  scaleType = "centerCrop",
}

src 能写什么

值同步？说明
res.drawable(name [, color]) / 其它 Drawable同步已着色的 Drawable 直接设到 ImageView
Bitmap同步设为图片位图
"@drawable/name" 或 "drawable/name"R 同步；工程位图异步先应用内 R.drawable（再试 ic_name），再工程 assets 图 / .lua
"@android:drawable/name"同步系统公开图标；无 copy / cut / paste 一类
文件路径 / URL异步由 Coil 加载

@drawable 怎么找图

  1. 应用资源：同名 R.drawable.name，没有再试 ic_name（与 res.drawable 相同）。
  2. 工程 assets：res/drawable/name.png（及 jpg / webp 等）。
  3. 工程脚本：res/drawable/name.lua 返回 Drawable / Bitmap。
  4. 仍没有 → 报错，不会静默空白。

命名：工具栏短名 save / search 可对应 ic_save.xml；资源文件本身已叫 ic_paste 时写 ic_paste 即可，不要写成 ic_ic_paste。

着色："@drawable/…" 不会自动 tint。需要颜色时用 res.drawable(name, color)，或再写 iconTint / TintColor。

padding / margin 与 Start·End

间距在属性扫完后统一写入（不会当成普通 setter 漏设）：

写法实现说明
padding 与 paddingLeft/Top/Right/BottomView.setPadding物理方向；单边可覆盖 padding
含 paddingStart 或 paddingEndView.setPaddingRelativeRTL 感知；缺省边回退 Left/Right/padding
layout_margin 与 Left/Top/Right/BottomLayoutParams.setMargins四边外边距
layout_marginStart / layout_marginEndsetMarginStart / setMarginEnd在 setMargins 之后写入，避免被覆盖
return {
  LinearLayout,
  orientation = "vertical",
  paddingStart = "16dp",
  paddingEnd = "16dp",
  {
    MaterialButton,
    text = "确定",
    layout_marginStart = "8dp",
    layout_marginEnd = "8dp",
  },
}

事件监听

所有以 on 开头的属性会按监听器处理。属性值可以是函数，也可以是绑定表中的函数名字符串。

local binding = {
	onButtonClick = function(view)
		print("clicked")
	end,
}

local view = loadlayout({
	Button,
	text = "点击",
	onClick = "onButtonClick",
}, binding)

{
	Button,
	text = "直接函数",
	onClick = function(view)
		print(view)
	end,
}

style / theme 用法

loadlayout 会在创建 View 时优先处理构造期样式字段，适合 MaterialButton、MaterialTextView、TextInputLayout、MaterialCardView 等组件。

兼容字段

  • style：旧写法；?attr/... 当 defStyleAttr；@style/... 或数字 style id 当 defStyleRes（四参构造，第三参为 0）。
  • theme：仅用于 ContextThemeWrapper，不与 style 资源混用。
  • styleAttr：构造函数 defStyleAttr（如 ?attr/materialButtonOutlinedStyle）。
  • styleRes：构造函数 defStyleRes；可与 styleAttr 组合；单独使用时走 (Context, attrs, 0, styleRes)。

Material 组件推荐优先 styleAttr；若要用具体 style 资源 id，写 styleRes（或数字 id），不要把 style id 当成第三参 attr。

支持的资源写法：

写法说明
?attr/name当前主题中的属性，例如 ?attr/materialButtonOutlinedStyle、?attr/colorPrimary
?android:attr/nameAndroid 系统 attr
@attr/name应用 attr 资源
@style/name应用 style 资源
@android:style/nameAndroid 系统 style
数字资源 ID直接使用 R.attr.xxx / R.style.xxx / MDC_R.attr.xxx

颜色属性

颜色字段推荐 "?attr/colorPrimary" 等（默认模板）。

{ MaterialTextView, textColor = "?attr/colorPrimary" }

示例

旧写法仍然可用：

import "com.google.android.material.button.MaterialButton"

return {
	MaterialButton,
	text = "轮廓按钮",
	style = "?attr/materialButtonOutlinedStyle",
}

推荐写法：显式区分 styleAttr 与 styleRes。

import "com.google.android.material.button.MaterialButton"

return {
	MaterialButton,
	text = "轮廓按钮",
	styleAttr = "?attr/materialButtonOutlinedStyle",
}

同时指定 theme overlay 与样式：

import "com.google.android.material.textfield.TextInputLayout"
import "com.google.android.material.textfield.TextInputEditText"

return {
	TextInputLayout,
	layout_width = "match",
	layout_height = "wrap",
	theme = "ThemeOverlay.Material3",
	styleAttr = "?attr/textInputOutlinedStyle",
	{
		TextInputEditText,
		layout_width = "match",
		layout_height = "wrap",
	}
}

如果已经拿到资源 ID，优先传数字 ID，可避免运行时字符串查找：

return {
	MaterialButton,
	text = "高性能写法",
	styleAttr = MDC_R.attr.materialButtonOutlinedStyle,
}

-- 纯 style 资源（IconButton 等）：styleRes / 数字 style id → 四参 defStyleRes
return {
	MaterialButton,
	styleRes = MaterialR.style.Widget_Material3_Button_IconButton,
	text = "",
}

说明

  • style 仍可用于既有布局。
  • Material 组件上 styleAttr / styleRes 较常见；也可再叠加 theme。
  • 字符串形式的资源引用会做缓存；列表项很多时，直接传数字资源 ID 通常更省开销。
  • 资源名无效时，运行日志里会给出较明确的提示，便于对照布局表排查。

CoordinatorLayout 特殊属性

loadlayout 对部分 Material/CoordinatorLayout 布局参数做了特殊处理：

属性说明示例
layout_behaviorCoordinatorLayout Behavior"@string/appbar_scrolling_view_behavior"
layout_anchor锚定到已定义 id 的 View"fab"
layout_scrollFlagsAppBar 滚动标志"scroll|enterAlways"
layout_collapseModeCollapsingToolbar 折叠模式"pin", "parallax"
layout_collapseParallaxMultiplier视差系数0.5

相对布局规则也支持字符串 id，例如 layout_below = "title"。被引用的 id 须在当前 View 之前定义。

补充说明

  • 第一个元素是 nil：多半还没 import 对应类，或类名写错。
  • id = "title" 会分配 Android ID，并把 View 放进绑定表（如 binding.title）。
  • style / styleAttr / styleRes / theme 只在构造 View 时生效。
  • padding* / layout_margin*（含 Start/End）在整表扫完后统一应用（见上文间距一节）。
  • 多数 Java setXxx 可用同名属性写，如 alpha = 0.5。MaterialButton 若默认 style 盖住 tint，可在 loadlayout 返回后再调一次 setBackgroundTintList。
  • 属性名对不上 setter / 字段时会打错误日志。专用键（id、padding*、src、pages、style*）走各自逻辑；onClick 等按监听器约定绑定。
  • 枚举字符串按属性分表：gravity、inputType、imeOptions、visibility 等；多个 flag 用 |，如 gravity = "center|end"。
  • textSize：带单位按单位换算；纯数字按 sp。宽高支持 match / wrap / fill 与尺寸字符串。
  • 子节点可以是嵌套表，也可以是字符串路径（会先 require 再加载）。
